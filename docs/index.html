my-agent-skills

Cold-start continuity for multi-session agent work.

Four installable skills that decide, write, check, and drive — so the next session picks up from a handoff, not a forgotten chat.

Install View on GitHub

Lifecycle

Decide → write → check → drive

  1. Decide

    start-interview

    Resolve forks before anything is written.

  2. Write

    plan-build

    Crystallize decisions into a cold-start HANDOFF tree.

  3. Check

    review-plan

    Gate the plan against quality while fixes are still cheap.

  4. Drive

    orchestrate-build

    Optional chunk-by-chunk build with cold-start subagents.

You rarely need all four. They are designed to flow in this order — and to survive a new chat, a new agent, or a week away — because continuity lives in files, not in-context memory.

Throughline

How they connect

start-interview resolves the decision tree → plan-build crystallizes it into a durable HANDOFF.md tree → review-plan gates that plan → orchestrate-build executes it chunk-by-chunk, reusing the same artifacts.

Every handoff between phases and sessions is explicit and cold-start-resumable. The unit of continuity is HANDOFF.md, not an agent’s chat history.

The loop closes with project-local lessons.md. A periodic, human-gated harvest turns recurring lessons into proposed suite edits — never automatic self-edits to the control plane.

Skills

Four peers, one lifecycle

  • start-interview

    Decide

    What
    Interviews you relentlessly about a plan or design, one decision at a time, with a recommended answer for each.
    When
    Before anything is written — when requirements are vague or a design has unresolved forks.
    Why
    Converts a fuzzy idea into resolved decisions, then hands off into plan mode / plan-build.
    How to start
    Invoke /start-interview (or ask to stress-test a plan or design).
    SKILL.md →

  • plan-build

    Write & maintain

    What
    Creates and maintains a durable handoff brief tree under docs/plans/<effort-slug>/, with HANDOFF.md as the entry point.
    When
    After decisions are locked — when you need artifacts any cold-start agent can resume from.
    Why
    Continuity rides on an explicit handoff, not an injected pointer or leftover chat context.
    How to start
    Ask to stand up or refresh a plan / handoff for the effort.
    SKILL.md →

  • review-plan

    Check

    What
    Two-phase plan review against quality gates; applies only what you approve, then re-reviews. Includes a resumability lens and reference checks.
    When
    On a plan-build plan (or any plan) before building.
    Why
    Catch broken references, scope creep, and unclear steps while they are still cheap to fix.
    How to start
    Ask to review the plan, triage feedback, or check it against the gates.
    SKILL.md →

  • orchestrate-build

    Drive

    optional

    What
    Breaks an approved plan into chunks and dispatches each to a serialized cold-start subagent, with re-verify, checkpoint, and escape hatches.
    When
    Once an approved plan-build tree exists and you want chunk-by-chunk build automated.
    Why
    Adds the bus and dispatch loop without changing the plan artifacts — opt-in once a plan exists.
    How to start
    Point it at an approved plan tree when you are ready to drive the build.
    SKILL.md →

Install

Symlink into your skills path

Each skill installs as a symlink so edits in this repo take effect immediately.

cd /path/to/my-agent-skills
for s in start-interview plan-build review-plan orchestrate-build; do
  ln -s "$PWD/$s" ~/.agents/skills/"$s"
  # optionally also: ln -s "$PWD/$s" ~/.cursor/skills/"$s"
done

Never place skills under ~/.cursor/skills-cursor/ — that directory is reserved for Cursor’s built-in skills.

Principles

How the suite stays reliable

  • Execution reliability over prose

    Crystallize deterministic procedure into scripts; degrade gracefully where execution is unavailable.

  • Explicit handoff entry

    Continuity rides on an explicit HANDOFF.md, not on an injected AGENTS.md pointer.

  • Human-gated by default

    Commits, scope changes, and any self-improvement of the skills require explicit user approval.

Projects accrete lessons.md; the suite improves only via reviewed proposals — never automatic self-edits.
